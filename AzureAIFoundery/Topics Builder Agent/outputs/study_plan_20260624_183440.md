# 🎯 Azure AI Engineer Study Plan
### Your Personalized 3-Month Roadmap

---

**📋 Plan Overview**

| Detail | Info |
|---|---|
| **Duration** | 12 Weeks (3 Months) |
| **Weekly Commitment** | 8–10 hours |
| **Total Study Hours** | ~96–120 hours |
| **Starting Point** | Basic Python + Azure VMs experience |
| **Goal** | Job-ready Azure AI Engineer |

---

> 💡 **How to use this plan:** Each week has a clear focus, specific modules to complete, and a simple hands-on task to reinforce what you learned. Modules are sequenced so each week builds on the last. Don't skip the hands-on tasks — they're the glue that makes everything stick.

---

## 📅 MONTH 1 — Build Your Foundation
### Weeks 1–4 | Azure Cloud Core + Python for AI

The first month is about filling the gaps. You've touched Azure before, so cloud concepts will feel familiar — but we'll make them solid. Python for AI is where you start building real engineering muscle.

---

### ✅ Week 1 — Azure Cloud Foundations
**Time:** ~8 hours | **Difficulty:** 🟢 Beginner

**Why this week matters:** Before building AI systems on Azure, you need to confidently navigate the platform, understand how resources are organized, and know how identity and security work. This is the scaffolding for everything ahead.

**Modules to complete:**

| # | Module | URL | Est. Time |
|---|---|---|---|
| 1 | Describe Azure Compute and Networking Services | [Open →](https://learn.microsoft.com/en-us/training/modules/describe-azure-compute-networking-services/) | 1.5 hrs |
| 2 | Describe Azure Storage Services | [Open →](https://learn.microsoft.com/en-us/training/modules/describe-azure-storage-services/) | 1 hr |
| 3 | Describe Core Architectural Components of Azure | [Open →](https://learn.microsoft.com/en-us/training/modules/describe-core-architectural-components-of-azure/) | 1.5 hrs |
| 4 | Describe Azure Identity, Access, and Security | [Open →](https://learn.microsoft.com/en-us/training/modules/describe-azure-identity-access-security/) | 1.5 hrs |

**🛠️ Hands-On Task:**
Log into the Azure Portal. Create a Resource Group, deploy a Storage Account inside it, and assign yourself a role using RBAC. Then delete everything. This simple loop — create, configure, secure, delete — is something you'll do hundreds of times as an AI Engineer.

**✔️ End of Week Checkpoint:**
- Can you explain what a Resource Group, Subscription, and Management Group are?
- Do you understand the difference between Authentication and Authorization in Azure?

---

### ✅ Week 2 — Azure Governance + Cost Awareness
**Time:** ~8 hours | **Difficulty:** 🟢 Beginner

**Why this week matters:** AI workloads can get expensive fast. Understanding how to govern resources and manage costs is a real professional skill that separates good engineers from great ones.

**Modules to complete:**

| # | Module | URL | Est. Time |
|---|---|---|---|
| 1 | Describe Cost Management in Azure | [Open →](https://learn.microsoft.com/en-us/training/modules/describe-cost-management-azure/) | 1.5 hrs |
| 2 | Describe Features and Tools for Governance and Compliance | [Open →](https://learn.microsoft.com/en-us/training/modules/describe-features-tools-azure-for-governance-compliance/) | 1.5 hrs |
| 3 | Set Up the VS Code Development Environment | [Open →](https://learn.microsoft.com/en-us/training/modules/set-up-vscode-dev-environment/) | 2 hrs |

**🛠️ Hands-On Task:**
Set up your local development environment completely this week. Install VS Code, configure the Python extension, install the Azure CLI, and log in to your Azure account from the terminal using `az login`. Then use the Azure Pricing Calculator to estimate the monthly cost of running a basic AI Vision API with 10,000 calls.

**✔️ End of Week Checkpoint:**
- Is your VS Code environment configured and connected to Azure?
- Can you use the Azure CLI to list your resource groups from the terminal?

### ✅ Week 3 — Python for AI: Data and ML Libraries
**Time:** ~9 hours | **Difficulty:** 🟡 Intermediate

**Why this week matters:** You know basic Python — now we level it up for AI work. NumPy, Pandas, and Scikit-learn are the building blocks of every ML workflow you'll encounter on Azure.

**Modules to complete:**

| # | Module | URL | Est. Time |
|---|---|---|---|
| 1 | Introduction to Machine Learning with Python and Scikit-learn | [Open →](https://learn.microsoft.com/en-us/training/paths/machine-learning-foundations-using-data-science/) | 4 hrs |

**🛠️ Hands-On Task:**
Using a free dataset (like the Titanic or Iris dataset from Kaggle or Scikit-learn's built-ins), write a Python script that:
1. Loads data with Pandas
2. Cleans and prepares it
3. Trains a simple classification model with Scikit-learn
4. Prints the accuracy score

Keep this script — you'll use it again when you move to Azure ML in Month 2.

**✔️ End of Week Checkpoint:**
- Can you load, inspect, and transform a DataFrame in Pandas?
- Can you train and evaluate a Scikit-learn model end-to-end?

---

### ✅ Week 4 — Python + Azure: REST APIs and SDKs
**Time:** ~9 hours | **Difficulty:** 🟡 Intermediate

**Why this week matters:** Almost every Azure AI service is called through a REST API or Python SDK. This week teaches you the exact pattern you'll use throughout the rest of this plan.

**Modules to complete:**

| # | Module | URL | Est. Time |
|---|---|---|---|
| 1 | Consume Azure AI Services | [Open →](https://learn.microsoft.com/en-us/training/modules/consume-azure-ai-services/) | 2.5 hrs |
| 2 | Build an NLP Solution with Azure AI Language (first 2 units) | [Open →](https://learn.microsoft.com/en-us/training/paths/build-natural-language-processing-solution/) | 2.5 hrs |

**🛠️ Hands-On Task:**
Create an Azure AI Services resource in the portal. Then write a Python script that calls the Text Analytics API using the REST endpoint and your API key — without using any SDK, just the `requests` library. Then rewrite the same call using the Azure SDK. Compare the two approaches and note what the SDK handles for you automatically.

**✔️ End of Week Checkpoint:**
- Do you understand the pattern of Endpoint + API Key + JSON payload for Azure AI calls?
- Can you read API documentation and implement a call in Python?

## 📅 MONTH 2 — Go Deep on Azure AI
### Weeks 5–9 | Azure AI Services + Azure Machine Learning

This is the core of your training. Month 2 is where you go from "cloud user" to "AI engineer." Expect more complexity, more hands-on work, and more confidence by the end.

---

### ✅ Week 5 — Azure AI Vision and Speech
**Time:** ~9 hours | **Difficulty:** 🟡 Intermediate

**Why this week matters:** Vision and Speech are among the most commonly deployed Azure AI services in real-world applications. Mastering them gives you practical, billable skills.

**Modules to complete:**

| # | Module | URL | Est. Time |
|---|---|---|---|
| 1 | Analyze Images with Azure AI Vision | [Open →](https://learn.microsoft.com/en-us/training/modules/analyze-images/) | 1.5 hrs |
| 2 | Read Text with Azure AI Vision (OCR) | [Open →](https://learn.microsoft.com/en-us/training/modules/read-text-images-documents-with-computer-vision-service/) | 1.5 hrs |
| 3 | Detect and Analyze Faces with Azure AI Vision | [Open →](https://learn.microsoft.com/en-us/training/modules/detect-analyze-faces/) | 1.5 hrs |
| 4 | Create Speech-Enabled Apps with Azure AI Speech | [Open →](https://learn.microsoft.com/en-us/training/modules/transcribe-speech-input-text/) | 1.5 hrs |
| 5 | Translate Speech with Azure AI Speech | [Open →](https://learn.microsoft.com/en-us/training/modules/translate-speech-speech-service/) | 1 hr |

**🛠️ Hands-On Task:**
Build a small Python script that takes an image file, sends it to Azure AI Vision, and prints out: (1) a description of the image, (2) any detected text via OCR. Bonus: Record a short audio clip and use the Speech-to-Text API to transcribe it.

**✔️ End of Week Checkpoint:**
- Can you deploy and call Azure AI Vision and Speech services from Python?
- Do you understand the difference between the Vision Read API and the standard analyze endpoint?

### ✅ Week 6 — Azure AI Language and Decision Services
**Time:** ~9 hours | **Difficulty:** 🟡 Intermediate

**Why this week matters:** Language AI is one of the fastest-growing areas in enterprise tech. Knowing how to extract meaning from text and build intelligent conversation flows is a core AI Engineer skill.

**Modules to complete:**

| # | Module | URL | Est. Time |
|---|---|---|---|
| 1 | Extract Insights from Text with Azure AI Language | [Open →](https://learn.microsoft.com/en-us/training/modules/extract-insights-text-with-text-analytics-service/) | 1.5 hrs |
| 2 | Build a Conversational Language Understanding Model | [Open →](https://learn.microsoft.com/en-us/training/modules/build-language-understanding-model/) | 2 hrs |
| 3 | Create a Bot with Azure AI Bot Service | [Open →](https://learn.microsoft.com/en-us/training/modules/design-bot-conversation-flow/) | 2 hrs |
| 4 | Detect Anomalies with Anomaly Detector | [Open →](https://learn.microsoft.com/en-us/training/modules/detect-anomalies-with-anomaly-detector/) | 1.5 hrs |

**🛠️ Hands-On Task:**
Use the Azure AI Language Studio (no code required) to create a Conversational Language Understanding project. Define 3 intents and 5+ example utterances each. Then call it from Python to classify a user's input. This mirrors exactly what you'd build for a real customer support bot.

**✔️ End of Week Checkpoint:**
- Can you explain the difference between Named Entity Recognition, Key Phrase Extraction, and Sentiment Analysis?
- Can you define intents and entities in the Language Studio?

### ✅ Week 7 — Azure Cognitive Search and Knowledge Mining
**Time:** ~9 hours | **Difficulty:** 🟡 Intermediate

**Why this week matters:** Cognitive Search is a key component of RAG (Retrieval-Augmented Generation) architectures — which you'll build in Month 3. Understanding it now will make the OpenAI section click much faster.

**Modules to complete:**

| # | Module | URL | Est. Time |
|---|---|---|---|
| 1 | Create an Azure Cognitive Search Solution | [Open →](https://learn.microsoft.com/en-us/training/modules/create-azure-cognitive-search-solution/) | 2 hrs |
| 2 | Implement Knowledge Mining with Azure Cognitive Search | [Open →](https://learn.microsoft.com/en-us/training/paths/implement-knowledge-mining-azure-cognitive-search/) | 4 hrs |

**🛠️ Hands-On Task:**
Upload a set of 5–10 text documents (can be Wikipedia pages you copy-paste) to Azure Blob Storage. Connect that storage to a Cognitive Search index. Run a search query from Python using the Azure Search SDK and retrieve results. Notice how the AI skillset automatically extracts key phrases and entities from your documents.

**✔️ End of Week Checkpoint:**
- Can you describe what an Index, Indexer, and Skillset are in Cognitive Search?
- Can you query a search index programmatically from Python?

**Why this week matters:** Azure ML is the professional platform for building, training, and managing ML models at scale. This week transitions you from calling pre-built AI APIs to building your own models.

**Modules to complete:**

**✔️ End of Week Checkpoint:**

### ✅ Week 9 — Azure ML: Pipelines, Deployment, and Monitoring
**Time:** ~10 hours | **Difficulty:** 🟠 Intermediate-Advanced

**Why this week matters:** Training models is only half the job. Deploying them reliably and monitoring them in production is what makes you a real AI Engineer, not just a data scientist.

**Modules to complete:**

| # | Module | URL | Est. Time |
|---|---|---|---|
| 1 | Run Pipelines in Azure Machine Learning | [Open →](https://learn.microsoft.com/en-us/training/modules/create-pipelines-in-aml/) | 2 hrs |
| 2 | Register an ML Model in Azure Machine Learning | [Open →](https://learn.microsoft.com/en-us/training/modules/register-and-deploy-model-with-amls/) | 1.5 hrs |
| 3 | Deploy a Model to a Managed Online Endpoint | [Open →](https://learn.microsoft.com/en-us/training/modules/deploy-model-managed-online-endpoint/) | 2 hrs |

**🛠️ Hands-On Task:**
Deploy the model you trained in Week 8 to a Managed Online Endpoint. Write a Python script that sends a POST request to your endpoint with sample data and prints the prediction. Then set up basic data drift monitoring. You now have a fully deployed ML model running on Azure — that's a real portfolio item.

**✔️ End of Week Checkpoint:**
- Can you explain the difference between real-time and batch inference endpoints?
- Do you understand what data drift is and why it matters in production?
